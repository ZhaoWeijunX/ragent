# 企业信息安全管理制度

> 版本：V2.3 | 生效日期：2025-01-01 | 归口部门：信息安全部  
> 配套文档：`数据分类分级规范.docx`、`等保合规自查报告.pdf`、`权限申请对照表.xlsx`

---

## 1. 总则

### 1.1 制定目的

为规范 **星云科技集团**（以下简称「集团」）及下属子公司、事业部的信息安全管理工作，保障信息系统与数据资产的 **机密性、完整性和可用性**（CIA 三元组），防范网络攻击、数据泄露与合规风险，依据《中华人民共和国网络安全法》《中华人民共和国数据安全法》《中华人民共和国个人信息保护法》及 GB/T 22239-2019《信息安全技术 网络安全等级保护基本要求》，制定本制度。

### 1.2 适用范围

本制度适用于：

- 集团总部及境内全资、控股子公司；
- 所有接入集团统一身份认证平台的业务系统（含 OA、ERP、CRM、数据中台、研发协作平台等）；
- 集团自有数据中心、公有云资源（阿里云、腾讯云、华为云）及托管机房；
- 与集团存在数据交换或系统对接的 **第三方供应商、外包人员及合作伙伴**。

### 1.3 管理原则

| 原则 | 说明 |
|------|------|
| **纵深防御** | 在网络、主机、应用、数据各层叠加安全控制，避免单点失效 |
| **最小权限** | 按岗位职责授予最低必要权限，定期复核与回收 |
| **默认拒绝** | 未明确授权的资源与操作一律禁止 |
| **职责分离** | 开发、测试、运维、审计角色分离，特权操作双人复核 |
| **可审计** | 关键操作留痕，日志保留不少于 **180 天**，高敏系统 **365 天** |
| **持续改进** | 以等保测评、渗透测试、红蓝对抗结果驱动整改闭环 |

## 2. 组织与职责

### 2.1 信息安全治理架构

```
董事会 / 信息安全委员会（决策层）
        │
信息安全部（管理协调层）
        │
   ┌────┼────┬────────┬────────┐
   │    │    │        │        │
 各事业部  IT运维  研发效能  内审合规  人力资源
（执行层）
```

### 2.2 岗位职责

| 角色 | 主要职责 |
|------|----------|
| **CISO（首席信息安全官）** | 制定安全战略与预算；向董事会汇报重大风险；审批 L3/L4 数据出境与特权账号 |
| **信息安全部** | 制度建设、等保测评、漏洞管理、安全事件协调、安全意识培训 |
| **IT 运维中心** | 基础设施加固、补丁管理、备份恢复、WAF/IDS 运维 |
| **研发效能部** | SDL（安全开发生命周期）、代码审计、依赖漏洞扫描（SCA） |
| **各业务系统 Owner** | 本系统数据分类分级、权限审批、业务连续性预案 |
| **全体员工** | 遵守制度、报告可疑行为、完成年度安全培训 |

### 2.3 安全联络人机制

各一级部门须指定 **信息安全联络人（Security Liaison）**，负责：

1. 传达集团安全政策与通知；
2. 协助本部门权限申请与账号复核；
3. 作为安全事件第一响应联系人（7×24 可联系）。

## 3. 资产与风险管理

### 3.1 信息资产登记

所有纳入等保范围的系统须在 **系统资产清单**（见 `系统资产清单.csv`）中登记，至少包含：

- 系统编码（如 `SYS-OA-001`）
- 系统名称、业务 Owner、技术负责人
- 部署环境（生产/预发/测试）
- 等保级别（二级/三级）
- 数据最高密级（L1～L4）
- 上次渗透测试日期

新增系统须在 **上线前 30 日** 完成资产登记与安全评审（Security Review）。

### 3.2 风险评估

| 活动 | 频率 | 负责方 |
|------|------|--------|
| 全集团风险评估 | 每年 1 次 | 信息安全部 |
| 等保测评（三级系统） | 每年 1 次 | 第三方测评机构 |
| 渗透测试（互联网暴露面） | 每半年 1 次 | 授权白帽/安全厂商 |
| 漏洞扫描（内网） | 每月 1 次 | IT 运维 |
| 依赖组件漏洞（SCA） | 每次发版前 | 研发效能 |

**风险处置时限**：

- **危急（CVSS ≥ 9.0 或已被在野利用）**：24 小时内临时缓解，72 小时内修复或下线
- **高危（CVSS 7.0～8.9）**：7 个工作日内修复
- **中危**：30 个工作日内修复
- **低危**：纳入版本计划，不超过 90 天

## 4. 访问控制与身份管理

### 4.1 统一身份认证

- 所有业务系统须接入集团 **SSO（OIDC/SAML）**，禁止独立本地账号（测试环境除外，须网络隔离）；
- 生产环境运维访问须经 **堡垒机（Jump Server）**，会话录屏保留 90 天；
- 特权账号（root、dba、域管）须启用 **MFA**，口令长度 ≥ 16 位，每 90 天强制轮换。

### 4.2 权限管理

权限申请须通过 ITSM 工单，对照 **权限申请对照表**（`权限申请对照表.xlsx`）选择角色模板：

| 审批层级 | 适用场景 |
|----------|----------|
| 直属经理 | L1/L2 数据只读权限 |
| 部门总监 + 系统 Owner | L3 数据读写、生产只读 |
| CISO 或授权委托人 | L4 数据、特权账号、数据库写权限 |
| 信息安全委员会 | 跨域数据导出、批量脱敏例外 |

**权限复核**：每季度由系统 Owner 发起「账号权限复核」，闲置账号（90 天未登录）自动冻结。

### 4.3 远程办公与 BYOD

- 访问内网资源须通过集团 **VPN + 终端合规检查**（EDR 在线、磁盘加密、系统补丁达标）；
- BYOD 设备不得存储 L3 及以上明文数据；
- 禁止使用个人网盘、即时通讯工具传输 L2 及以上业务文件。

## 5. 数据安全

### 5.1 分类分级

数据按业务影响分为 **L1（公开）～ L4（核心机密）** 四个等级，详细定义见 `数据分类分级规范.docx`。核心要求：

| 等级 | 典型数据 | 存储 | 传输 | 展示 |
|------|----------|------|------|------|
| L1 | 公司公告、产品白皮书 | 明文 | HTTPS | 无限制 |
| L2 | 内部制度、会议纪要 | 明文+访问控制 | HTTPS + 内网 | 禁止外发 |
| L3 | 客户 PII、合同、薪酬区间 | AES-256 字段加密 | TLS 1.2+ | 脱敏展示 |
| L4 | 身份证号、银行卡、源码、密钥 | HSM/国密加密 | 专线/mTLS | 最小可见+水印 |

### 5.2 数据生命周期

1. **采集**：明示告知、最小必要、获得合法授权；
2. **存储**：境内存储为原则，出境须安全评估与审批；
3. **使用**：按授权目的使用，禁止未授权二次加工；
4. **共享**：与第三方签订 DPA，明确安全责任与审计权；
5. **销毁**：存储介质报废须 **消磁或物理粉碎**，留存销毁记录 3 年。

### 5.3 个人信息保护

处理个人信息须满足：

- 制定个人信息保护影响评估（PIA）；
- 提供查询、更正、删除、撤回同意渠道；
- 发生个人信息泄露时，**72 小时内** 向监管部门报告并通知受影响个人（法律另有规定除外）。

## 6. 网络安全

### 6.1 网络分区

```
互联网区（DMZ）
    │ WAF / Anti-DDoS
应用服务区（APP Zone）
    │ 防火墙 + 微隔离
数据区（DB Zone）
    │ 禁止直连互联网
运维管理区（Mgmt Zone）
    │ 堡垒机唯一入口
```

### 6.2 边界防护

- 互联网入口部署 **WAF**，启用 OWASP Top 10 规则集；
- 办公网与生产网 **逻辑隔离**，跨区访问须走专线或零信任网关；
- 禁止生产数据库、中间件端口对办公网开放。

### 6.3 无线与访客网络

- 员工 Wi-Fi 与访客 Wi-Fi **物理或逻辑隔离**；
- 访客网络仅允许访问互联网，禁止访问内网段；
- 无线认证采用 WPA3-Enterprise + 802.1X。

## 7. 安全开发与运维

### 7.1 SDL 要求

| 阶段 | 安全活动 |
|------|----------|
| 需求 | 威胁建模（STRIDE）、数据分级标注 |
| 设计 | 安全架构评审、密钥管理方案 |
| 开发 | 安全编码规范、SAST 扫描 |
| 测试 | DAST、渗透测试、依赖漏洞扫描 |
| 发布 | 变更审批、配置基线核查 |
| 运维 | 日志监控、应急响应、补丁管理 |

### 7.2 变更与发布

- 生产变更须走 **CAB（变更顾问委员会）** 审批，紧急变更须事后 24 小时内补办；
- 禁止在生产环境直接调试，禁止将生产数据导入开发环境（须脱敏）；
- 容器镜像须经漏洞扫描，危急漏洞镜像禁止部署。

### 7.3 日志与监控

须集中采集并保护的日志类型：

- 身份认证（成功/失败/MFA）
- 权限变更与特权操作
- 数据导出与大批量查询
- 防火墙/WAF 拦截记录
- 堡垒机会话日志

SIEM 平台对 **暴力破解、异常登录地、大批量下载、权限提升** 等场景配置实时告警。

## 8. 第三方与供应链安全

### 8.1 供应商准入

采购涉及数据处理或系统对接的供应商前，须完成：

1. 安全资质审查（等保、ISO 27001、SOC 2 等）；
2. 签署《信息安全与保密协议》及《数据处理协议（DPA）》；
3. 明确数据归属、销毁义务与审计条款。

### 8.2 外包人员管理

- 外包账号须标注 `ext-` 前缀，权限不得超过同等岗位正式员工；
- 合同结束 **当日** 回收全部账号与物理门禁；
- 外包人员不得接触 L4 数据，特殊情况须 CISO 书面批准。

## 9. 安全事件管理

### 9.1 事件分级

| 级别 | 定义 | 示例 | 响应时限 |
|------|------|------|----------|
| **P0 特别重大** | 核心系统瘫痪或大量 L4 数据泄露 | 勒索软件加密生产库、10 万+个人信息泄露 | 15 分钟启动应急 |
| **P1 重大** | 重要系统受损或有限 L3/L4 泄露 | 单系统入侵、高管账号失陷 | 30 分钟 |
| **P2 较大** | 局部影响，无确认数据泄露 | 钓鱼邮件、Web 漏洞被扫描 | 2 小时 |
| **P3 一般** | 潜在风险，已自动阻断 | WAF 拦截、暴力破解失败 | 1 工作日 |

### 9.2 应急响应流程

详见 `应急响应流程图.png`，通报格式参照 `安全事件通报模板.txt`。

**黄金四小时**（P0/P1）：

1. **0～15 分钟**：发现人报告 SOC，启动应急群，初步定级；
2. **15～60 分钟**：隔离受影响资产，保全日志与镜像，遏制扩散；
3. **1～4 小时**：根因分析、修复方案、是否监管报告评估；
4. **4～24 小时**：对外通报（如需）、业务恢复、临时加固；
5. **7 日内**：完成事件报告与复盘，更新预案。

### 9.3 禁止行为

- 擅自对外披露安全事件细节；
- 在未授权情况下联系攻击者；
- 破坏或覆盖与事件相关的日志证据。

## 10. 业务连续性与灾难恢复

### 10.1 备份策略

| 数据类型 | RPO | RTO | 备份方式 |
|----------|-----|-----|----------|
| 核心交易库 | ≤ 15 分钟 | ≤ 1 小时 | 实时同步 + 每日全备 |
| 一般业务库 | ≤ 1 小时 | ≤ 4 小时 | 每小时增量 |
| 文件存储 | ≤ 4 小时 | ≤ 8 小时 | 跨区域复制 |
| 配置与代码 | ≤ 24 小时 | ≤ 2 小时 | Git + 配置仓库 |

### 10.2 演练要求

- 核心系统 **每半年** 进行一次灾难恢复演练；
- 演练须形成记录，包括恢复时长、数据一致性验证与改进项。

## 12. 违规处理

违反本制度的行为，按情节轻重给予：

- 口头警告、书面警告、记过、解除劳动合同；
- 涉嫌违法犯罪的，移送司法机关；
- 造成经济损失的，依法追究赔偿责任。

**零容忍行为**（直接解除劳动合同）：

- 故意泄露、出售 L4 数据或用户个人信息；
- 绕过安全控制访问未授权系统；
- 在生产环境植入恶意代码或后门。

### 13.3 FAQ

**Q1：员工忘记密码怎么办？**  
A：通过 SSO 自助重置，须 MFA 验证。连续 5 次失败锁定 30 分钟，联系 IT 服务台解冻。

**Q2：开发环境可以使用生产数据吗？**  
A：禁止。须使用脱敏工具（集团统一平台 `mask.xingyun.internal`）处理后方可导入。

**Q3：发现可疑钓鱼邮件如何处理？**  
A：不要点击链接或附件，转发至 `phishing@xingyun.com`，并报告部门安全联络人。

**Q4：个人笔记本电脑丢失怎么办？**  
A：立即报告 IT 服务台与信息安全部，远程擦除设备（如已装 EDR），并修改全部相关密码。

**Q5：等保三级系统需要满足哪些核心要求？**  
A：详见 `等保合规自查报告.pdf`；本制度第 4～7 章为日常执行层面的落地要求。
